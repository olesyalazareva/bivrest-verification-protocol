
# BIVREST — протокол групповой оценки ролевого поведения

**Версия:** 1.0

## 1**Проблема:** В ролевых играх и исследовательских сессиях сложно объективно оценить качество отыгрыша. Оценка ведущего субъективна, у участников нет единого критерия, данные сложно структурировать и сравнивать.

Обычные методы сбора данных (интервью, опросы, фокус-группы) дают искажённые результаты из-за социальной желательности. Участники говорят то, что «принято», а не то, что думают на самом деле.

**Что предлагает BIVREST:** Единый механизм оценки через групповое голосование. Группа голосует «Верю / Не верю» после каждого действия. Вердикт выносится по большинству. Результат фиксируется в структурированном логе.

**Этот протокол предназначен для сбора данных в ситуациях, где обычные методы (интервью, опросы) дают искажённые результаты из-за социальной желательности.** Ролевая игра позволяет снизить этот эффект, так как участник действует от лица персонажа, а не себя. 

## 2. BIVREST — это протокол сбора данных через ролевую игру. В его основе лежит механизм бинарного голосования **«Верю / Не верю»**.

Протокол фиксирует:
- действия участников
- результаты голосования группы
- итоговый вердикт

Всё это сохраняется в структурированном логе.


## 3. Для кого BIVREST

| Аудитория | Зачем |
|---|---|
| **Исследователи** | Сбор структурированных данных о ролевом поведении в условиях, где социальная желательность мешает получить достоверные ответы |
| **Психологи** | Фиксация групповой динамики и моментов выхода из роли |
| **Геймдизайнеры** | Готовая механика оценки отыгрыша для встраивания в игру |
| **AI-инженеры** | Протокол сбора размеченных данных для обучения моделей |


**Расшифровка названия:**

| Буква | Значение | Описание |
|---|---|---|
| **B** | Bifurcation | Разрыв между «Я» и аватаром |
| **I** | Inversion | Действие через противоположную роль |
| **V** | Verification | Механизм «Верю / Не верю» |
| **R** | Role | Аватар как защитный экран |
| **E** | Empty | Деконструкция через «Пустой стул» |
| **S** | Script | Сценарий, подлежащий деконструкции |
| **T** | Therapy | Интеграция нового сценария |

**Статус:** Задепонирован в НРИС.


### 3.1. Ключевые отличия BIVREST от аналогов

| Параметр | BIVREST | Психодрама / КПТ / Фокус-группы |
|---|---|---|
| **Аватар** | Максимально отличен от себя | Роль близка к себе или отсутствует |
| **Верификация** | Бинарное голосование группы | Авторитарный вердикт эксперта / ведущего |
| **Штраф** | За несоответствие роли — потеря ресурса | Нет штрафа — можно отыгрывать неубедительно |
| **Пустой стул** | При повторном «Не верю» — выход из роли | Требуется присутствие второго человека (в психодраме) |
| **Смена роли** | Принудительная (через механику игры) | Можно бесконечно прятаться за маской |
| **Формат данных** | Структурированный лог (JSON/CSV) | Качественное описание, сложно структурировать |

## 4. Как BIVREST работает

### 4.1. Участники протокола

| Роль | Описание |
|---|---|
| **Группа** | Минимум 3 человека, голосуют после каждого действия |
| **Наблюдатель** | Фиксирует результаты голосования и ведёт лог |
| **Фасилитатор** | Управляет сессией, выдаёт задания (опционально) |

### 4.2. Критерий голосования

BIVREST не предписывает конкретный критерий — он задаётся перед началом работы.

**Примеры критериев:**
- «Это действие соответствует роли?»
- «Это выглядит убедительно?»
- «Это правдиво в контексте игры?»

Группа голосует «Верю / Не верю» относительно заданного критерия.

### 4.3. Стадии работы протокола

| Буква | Стадия | Действие |
|---|---|---|
| **B** | Believe / Don't Believe | Группа голосует по заданному критерию |
| **I** | Information | Предъявление информации для оценки |
| **V** | Voting | Каждый участник отдаёт голос |
| **R** | Record | Фиксация результатов голосования |
| **E** | Evaluate | Определение вердикта по большинству |
| **S** | Store | Сохранение структурированного лога |
| **T** | Transfer | Передача данных для анализа |

### 4.4. Полный цикл одной итерации
Действие участника → Голосование группы → Фиксация результата → Лог



Каждая итерация даёт одну запись в логе.



## 5. Реализация BIVREST в игре UNIVERSE 69

UNIVERSE 69 — это игра, в которую BIVREST встроен как штатный механизм. Полные правила, карты и роли находятся в отдельном репозитории:

**→ [UNIVERSE 69 на GitHub](https://github.com/olesyalazareva/universe69)**

### 5.1. Как BIVREST встроен в механику UNIVERSE 69

| Механика UNIVERSE 69 | Роль в BIVREST |
|---|---|
| **Карты с заданиями** | Генерируют информацию для оценки (стадия I — Information) |
| **Аватары (роли)** | Задают контекст, в котором оценивается действие |
| **Голосование «Верю / Не верю»** | Основной механизм верификации (стадия B — Believe) |
| **Фишки «Шанс»** | Ресурс, который теряется при «Не верю» (механизм санкции) |
| **«Пустой стул»** | Выход из роли при повторном «Не верю» (стадия E — Empty) |
| **Логирование** | Фиксация всех действий и голосов (стадии R, E, S) |

### 5.2. Игровой цикл с BIVREST
Игрок выбирает аватар (роль)

Игрок получает карту с заданием

Игрок выполняет задание от лица аватара (отыгрыш)

Группа голосует «Верю» или «Не верю»

Если «Не верю» — игрок теряет фишку «Шанс»

Если два «Не верю» подряд — игрок выходит из роли («Пустой стул»)

Результаты записываются в лог



### 5.3. Что даёт эта связка

| Для игры | Для исследования |
|---|---|
| Чёткий критерий оценки отыгрыша | Структурированные данные |
| Объективность через групповое решение | Бинарная разметка для обучения моделей |
| Штраф за неубедительный отыгрыш | Фиксация моментов срыва роли |
| Механика выхода из роли | Данные о групповой динамике |

### 5.4. Формат лога в UNIVERSE 69

| Поле | Тип | Описание |
|---|---|---|
| `session_id` | string | Идентификатор сессии |
| `round` | integer | Номер раунда |
| `player_id` | string | Идентификатор игрока |
| `avatar` | string | Роль игрока |
| `task` | string | Задание с карты |
| `action_text` | text | Отыгрыш (что сказал/сделал) |
| `votes_believe` | integer | Количество голосов «Верю» |
| `votes_dont_believe` | integer | Количество голосов «Не верю» |
| `verdict` | string | «Верю» или «Не верю» |
| `tokens_lost` | integer | Сколько фишек потеряно |
| `empty_chair` | boolean | Был ли выход из роли |




BIVREST может использоваться и с другими ролевыми системами, но в UNIVERSE 69 он реализован как готовая механика.



## 6. Ограничения

- Результат голосования отражает мнение группы, а не объективную истину
- Протокол не гарантирует согласованность оценок между разными группами
- Требуется несколько сессий для получения статистически значимых данных
- Готовый датасет не предоставляется
