## Проблема

В ролевых играх и исследовательских сессиях сложно оценить, насколько поведение участника **соответствует его роли**. Психологи называют это **конгруэнтностью** — согласованностью между действиями и ожиданиями, связанными с ролью.

**С чем сталкиваются исследователи и фасилитаторы:**

| Метод | Проблема |
|---|---|
| **Экспертная оценка** | Субъективна, зависит от опыта и предпочтений ведущего |
| **Опросы участников** | Социальная желательность — участники говорят то, что «принято», а не то, что думают |
| **Самооценка** | Не отражает восприятие группы, расхождение между "я так хотел сыграть" и "меня так восприняли" |
| **Свободное наблюдение** | Данные сложно структурировать, сравнивать и анализировать |

**В результате:** нет единого критерия, данные несопоставимы, а выводы строятся на субъективных впечатлениях.

## Что предлагает BIVREST

BIVREST — это **протокол сбора данных**, который стандартизирует процесс оценки ролевого поведения.

**Основная механика:**

Группа голосует **«Верю / Не верю»** после каждого действия. Критерий голосования **задаётся перед началом сессии** и может быть, например:

> *«Это действие соответствует роли?»*

Голоса подсчитываются, вердикт выносится по большинству, результат фиксируется в структурированном логе.

**Ключевая идея:**

Вместо того чтобы спрашивать участника: *"Как ты себя чувствовал в роли?"* — мы спрашиваем группу: *"Было ли это конгруэнтно роли?"*

Это **смещает фокус** с самооценки на **социальное восприятие** и даёт данные, которые можно анализировать статистически.

## Что BIVREST измеряет на самом деле

**Важно понимать:** BIVREST не претендует на измерение «объективной истины». Он измеряет **восприятие группы**.

| Что BIVREST даёт | Что BIVREST НЕ даёт |
|---|---|
| Социально-валидированную оценку конгруэнтности | Объективную истину о качестве отыгрыша |
| Структурированные данные для анализа | Готовый ответ на вопрос «хорошо ли сыграно?» |
| Возможность сравнивать сессии при фиксированном критерии | Сравнение между разными критериями |
| Данные о групповой динамике и социальных ожиданиях | Индивидуальную психологическую диагностику |

**BIVREST — это инструмент для исследования того, как группа воспринимает ролевое поведение.** 

Он полезен, если ваш исследовательский вопрос звучит так:
- «Какие действия группа считает конгруэнтными роли?»
- «Как меняется восприятие роли со временем?»
- «Что вызывает рассогласование между ожиданиями и действиями?»

## Для кого BIVREST

| Аудитория | Зачем |
|---|---|
| **Исследователи** | Сбор структурированных данных о ролевом поведении и групповом восприятии |
| **Психологи** | Фиксация групповой динамики, исследование социальных ожиданий |
| **Геймдизайнеры** | Готовая механика оценки отыгрыша для встраивания в игру |
| **AI-инженеры** | Протокол сбора размеченных данных для обучения моделей распознаванию конгруэнтности |

**Расшифровка названия:**

| Буква | Значение | Описание |
|---|---|---|
| **B** | Believe / Don't Believe | Механизм бинарного голосования |
| **I** | Information | Предъявление информации для оценки |
| **V** | Voting | Голосование группы |
| **R** | Record | Фиксация результатов |
| **E** | Evaluate | Определение вердикта по большинству |
| **S** | Store | Сохранение структурированного лога |
| **T** | Transfer | Передача данных для анализа |

## Как работает протокол

### Участники протокола

| Роль | Описание |
|---|---|
| **Группа** | Минимум 3 человека, голосуют после каждого действия |
| **Наблюдатель** | Фиксирует результаты голосования и ведёт лог |
| **Фасилитатор** | Управляет сессией, выдаёт задания (опционально) |

### Критерий голосования

Критерий **задаётся перед сессией** и остаётся неизменным на протяжении всей сессии.

**Рекомендуемый критерий для исследований конгруэнтности:**

> *«Это действие соответствует роли?»*

**Другие возможные критерии (для других исследовательских задач):**

- «Это выглядит убедительно?»
- «Это правдиво в контексте игры?»
- «Я верю, что персонаж так поступил бы?»

### Стадии работы протокола

| Буква | Стадия | Действие |
|---|---|---|
| **B** | Believe / Don't Believe | Группа голосует по заданному критерию |
| **I** | Information | Участник предъявляет действие (отыгрыш) |
| **V** | Voting | Каждый участник отдаёт голос («Верю» или «Не верю») |
| **R** | Record | Наблюдатель фиксирует результаты |
| **E** | Evaluate | Вердикт определяется по большинству |
| **S** | Store | Данные сохраняются в структурированном логе |
| **T** | Transfer | Данные передаются для анализа |

### Полный цикл одной итерации

**Действие участника → Голосование группы → Фиксация результата → Запись в лог**

Каждая итерация — одна строка в логе.

## Структура данных

### Формат лога

| Поле | Тип | Описание |
|---|---|---|
| `session_id` | string | Идентификатор сессии |
| `round` | integer | Номер раунда |
| `player_id` | string | Идентификатор игрока |
| `avatar` | string | Роль игрока |
| `task` | string | Задание с карты |
| `action_text` | text | Отыгрыш (что сказал/сделал) |
| `votes_believe` | integer | Количество голосов «Верю» |
| `votes_dont_believe` | integer | Количество голосов «Не верю» |
| `verdict` | string | «Верю» или «Не верю» |
| `tokens_lost` | integer | Сколько фишек потеряно (если используется штраф) |
| `empty_chair` | boolean | Был ли выход из роли |

## 🚀 Быстрый старт

### 1. Скачайте описание протокола
В этом репозитории есть всё необходимое для проведения исследования.

### 2. Прочитайте ключевые документы
- **[Техническая спецификация](protocol_specification.md)** — детальные правила и форматы данных
- **[Руководство по аннотации](annotation_guide.md)** — для разметки данных под AI-задачи
- **[Пример лога](session_example.txt)** — как выглядит заполненная запись

### 3. Проведите сессию

1. **Соберите группу** — минимум 3 человека
2. **Определите критерий** — например: «Это действие соответствует роли?»
3. **Проведите серию итераций** — 5–20 раундов
4. **Фиксируйте результаты** — заполняйте лог после каждого действия

### 4. Анализируйте данные

Полученный лог даёт возможность:

- **Количественного анализа** — частоты, распределения, динамика
- **Сравнительного анализа** — между разными ролями и игроками
- **Обучения моделей** — если вы размечаете данные для AI

**Если вам нужна консультация по применению метода** — напишите автору (контакты внизу).

## Реализация в игре UNIVERSE 69

UNIVERSE 69 — это игра, в которую BIVREST встроен как штатный механизм. Полные правила, карты и роли находятся в отдельном репозитории:

**→ [UNIVERSE 69 на GitHub](https://github.com/olesyalazareva/universe-69)**

### Как BIVREST встроен в механику UNIVERSE 69

| Механика UNIVERSE 69 | Роль в BIVREST |
|---|---|
| **Карты с заданиями** | Генерируют информацию для оценки (стадия I) |
| **Аватары (роли)** | Задают контекст для оценки конгруэнтности |
| **Голосование «Верю / Не верю»** | Основной механизм верификации (стадия B) |
| **Фишки «Шанс»** | Ресурс, который теряется при «Не верю» |
| **«Пустой стул»** | Выход из роли при повторном «Не верю» |
| **Логирование** | Фиксация всех действий и голосов |

BIVREST может использоваться с любыми ролевыми системами — UNIVERSE 69 является примером готовой реализации.

## Ограничения

- **Групповая оценка ≠ объективная истина** — результат отражает мнение группы, а не абсолютную реальность
- **Критерий должен быть фиксирован** — если вы меняете критерий, данные становятся несопоставимыми
- **Требуется несколько сессий** — для статистически значимых данных
- **Социальная желательность не исчезает** — она трансформируется в желание соответствовать ожиданиям группы
