
# Техническая спецификация BIVREST

**Версия:** 1.0  
**Дата:** 2026  
**Статус:** Задепонирован в НРИС

---

## 1. Назначение документа

Настоящий документ содержит техническое описание протокола групповой оценки ролевого поведения **BIVREST** (Believe/Don't Believe — Information — Voting — Record — Evaluate — Store — Transfer).

Документ предназначен для:
- исследователей, внедряющих протокол в свои эксперименты
- разработчиков, интегрирующих BIVREST в игровые системы
- AI-инженеров, собирающих данные для обучения моделей
- фасилитаторов, проводящих сессии по протоколу

**Основная задача протокола:** сбор структурированных данных о групповом восприятии конгруэнтности ролевого поведения.

---

## 2. Основные понятия

| Термин | Определение |
|---|---|
| **Протокол** | Стандартизированная процедура сбора данных |
| **Конгруэнтность** | Соответствие поведения участника ожиданиям, связанным с его ролью |
| **Итерация** | Один полный цикл: действие → голосование → фиксация → лог |
| **Лог** | Структурированная запись всех итераций сессии |
| **Вердикт** | Решение группы по итогам голосования («Верю» или «Не верю») |
| **Критерий** | Правило, по которому группа оценивает действие (задаётся перед сессией) |
| **Социальная валидация** | Подтверждение оценки через коллективное решение группы |

## 3. Роли и обязанности

### 3.1. Группа (минимум 3 человека)

**Обязанности:**
- Голосовать после каждого действия по заданному критерию
- Обосновывать своё решение (опционально, для качественного анализа)
- Соблюдать конфиденциальность данных

**Требования:**
- Все члены группы должны быть ознакомлены с критерием голосования
- Голосование осуществляется одновременно (тайное или открытое — определяется исследователем)
- Каждый участник голосует один раз за итерацию

### 3.2. Наблюдатель (может совпадать с фасилитатором)

**Обязанности:**
- Фиксировать результаты голосования в лог
- Записывать действие участника (текст отыгрыша)
- Отслеживать количество голосов «Верю» и «Не верю»
- Отмечать особые события (выход из роли, смена роли и т.д.)

**Требования:**
- Внимательность и точность
- Знание структуры лога
- Нейтральность (не влияет на голосование)

### 3.3. Фасилитатор (опционально)

**Обязанности:**
- Выдавать задания участникам
- Следить за соблюдением временного регламента
- Управлять процессом голосования
- Разрешать спорные ситуации

**Требования:**
- Знание протокола
- Навыки управления группой
- Беспристрастность

## 4. Критерий голосования

### 4.1. Определение

Критерий — это вопрос или утверждение, относительно которого группа оценивает действие участника.

**Критерий задаётся ДО начала сессии** и должен быть:
- понятен всем участникам
- однозначен
- применим ко всем действиям в сессии
- **фиксирован** на протяжении всей сессии

### 4.2. Рекомендуемый критерий для исследований конгруэнтности

> **«Это действие соответствует роли?»**

Этот критерий позволяет измерять **групповое восприятие конгруэнтности** — насколько действия участника соответствуют ожиданиям группы от его роли.

### 4.3. Альтернативные критерии (для других исследовательских задач)

| Критерий | Для каких исследований |
|---|---|
| «Это выглядит убедительно?» | Исследования доверия и убедительности |
| «Это правдиво в контексте игры?» | Исследования правдоподобия |
| «Я верю, что персонаж так поступил бы?» | Исследования эмпатии и вовлечённости |

### 4.4. Требования к критерию

- **Релевантность:** критерий должен быть связан с исследовательским вопросом
- **Чёткость:** все участники должны одинаково понимать критерий
- **Стабильность:** критерий не меняется в течение всей сессии
- **Бинарность:** ответ может быть только «Верю» или «Не верю»

## 5. Процедура голосования

### 5.1. Алгоритм полной итерации

1. **Предъявление действия** (стадия I — Information)
   - Участник выполняет действие (говорит или делает что-то от лица роли)
   - Наблюдатель фиксирует действие в лог

2. **Голосование** (стадия B — Believe / Don't Believe)
   - Группа голосует по заданному критерию
   - Каждый участник отдаёт один голос («Верю» или «Не верю»)

3. **Подсчёт голосов** (стадия V — Voting)
   - Наблюдатель подсчитывает количество голосов «Верю» и «Не верю»
   - Результаты заносятся в лог

4. **Определение вердикта** (стадия E — Evaluate)
   - Вердикт определяется по большинству голосов
   - Если голосов поровну — вердикт «Не верю» (по умолчанию)

5. **Фиксация результата** (стадия R — Record)
   - Результаты записываются в лог
   - Дополнительные примечания (по желанию)

6. **Сохранение** (стадия S — Store)
   - Лог сохраняется в выбранном формате (JSON, CSV)

7. **Передача** (стадия T — Transfer)
   - Данные подготавливаются для анализа

### 5.2. Правила голосования

| Правило | Описание |
|---|---|
| **Один голос** | Каждый участник голосует один раз за итерацию |
| **Без воздержания** | Участник обязан выбрать «Верю» или «Не верю» |
| **Синхронность** | Все голоса подаются одновременно (или в течение короткого интервала) |
| **Итог по большинству** | Вердикт определяется простым большинством |

### 5.3. Особые случаи

| Случай | Решение |
|---|---|
| **Равенство голосов** | Вердикт = «Не верю» (консервативное решение) |
| **Один участник не голосует** | Голос не учитывается, но результат принимается |
| **Два «Не верю» подряд** | Инициируется выход из роли («Пустой стул») — опциональная механика |

## 6. Структура лога

### 6.1. Поля и типы данных

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `session_id` | string | Да | Уникальный идентификатор сессии |
| `round` | integer | Да | Номер раунда (начинается с 1) |
| `player_id` | string | Да | Уникальный идентификатор игрока |
| `avatar` | string | Да | Название роли (аватара) |
| `task` | string | Да | Задание, полученное игроком |
| `action_text` | text | Да | Текст отыгрыша (что сказал/сделал) |
| `votes_believe` | integer | Да | Количество голосов «Верю» |
| `votes_dont_believe` | integer | Да | Количество голосов «Не верю» |
| `verdict` | string | Да | «Верю» или «Не верю» |
| `tokens_lost` | integer | Да | Количество потерянных фишек (0, если не используется) |
| `empty_chair` | boolean | Да | Был ли выход из роли (true/false) |
| `timestamp` | string | Нет | Время записи (ISO 8601) |
| `notes` | text | Нет | Дополнительные примечания |

6.3. Правила заполнения
Поле	Правило
session_id	Формат: [префикс]_[дата]_[номер], например: bivrest_20250625_001
round	Начинается с 1, увеличивается на 1 после каждой итерации
player_id	Уникальный для каждого игрока в сессии
avatar	Полное название роли, без сокращений
task	Текст задания, как он был сформулирован
action_text	Дословная запись того, что сказал/сделал игрок
votes_believe	Целое число, сумма votes_believe + votes_dont_believe = размер группы
verdict	Всегда «Верю» или «Не верю»
empty_chair	true только если инициирован выход из роли
7. Требования к сессии
7.1. Минимальные требования
Параметр	Значение
Количество участников в группе	Минимум 3 человека
Наблюдатель	1 человек (может совпадать с фасилитатором)
Длительность одной итерации	Не более 5 минут
Количество итераций	От 5 до 20 на сессию
Язык проведения	Определяется исследователем
Фиксация данных	Обязательна для каждой итерации
Критерий голосования	Фиксирован на всю сессию
7.2. Рекомендации
Проводить пилотную сессию для отладки процедуры

Записывать аудио (с согласия участников) для верификации

Использовать таймер для контроля времени

Поощрять участников за вовлечённость

Чётко объяснить критерий голосования перед началом

8. Обработка данных
8.1. Предобработка
Проверка на пропуски в данных

Удаление технических ошибок (некорректные форматы)

Анонимизация (удаление имён и идентифицирующей информации)

Преобразование в формат для анализа (CSV, Excel, R, Python)

8.2. Типы анализа
Тип анализа	Что измеряется
Частотный	Распределение голосов «Верю» / «Не верю»
Временной	Динамика изменения вердиктов по раундам
Групповой	Согласованность группы (коэффициент конкордации)
Индивидуальный	Успешность конкретных игроков в поддержании конгруэнтности
Контекстный	Зависимость оценки от типа задания или роли
8.3. Метрики
Метрика	Описание	Формула
Успешность (S)	Доля «Верю» в сессии (оценка конгруэнтности)	S = votes_believe / (votes_believe + votes_dont_believe)
Согласованность (C)	Доля единогласных решений	C = unanimous_rounds / total_rounds
Стабильность (St)	Изменчивость вердиктов по раундам	St = 1 - (изменения_вердикта / (total_rounds - 1))
8.4. Интерпретация данных
Высокий S (> 0.7): группа воспринимает действия как конгруэнтные роли

Низкий S (< 0.4): группа фиксирует рассогласование

Высокая C (> 0.8): высокая согласованность в группе

Низкая C (< 0.5): мнения группы разделились — возможен спорный критерий или неоднозначная роль

9. Интеграция с AI/ML
9.1. Формат для обучения моделей
BIVREST-логи являются размеченными данными для задач классификации:

Вход (features): action_text, avatar, task, context

Целевая переменная (label): verdict («Верю» / «Не верю»)

Пример задачи: Модель учится предсказывать, будет ли действие оценено группой как конгруэнтное роли, на основе текста отыгрыша.

9.2. Рекомендации по аннотации
Использовать структурированные данные (CSV/JSON)

Проверять согласованность между разными группами

Исключать случаи с нарушением процедуры

Сохранять контекстные метаданные (роль, задание)

9.3. Формат для датасета
Колонка	Описание
text	Текст отыгрыша
role	Роль игрока
task	Задание
label	believe (конгруэнтно) или dont_believe (не конгруэнтно)
9.4. Потенциальные применения
Обучение LLM распознаванию конгруэнтности в тексте

Создание классификатора для оценки отыгрыша в реальном времени

Исследование факторов, влияющих на восприятие конгруэнтности
