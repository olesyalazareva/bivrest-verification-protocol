1. Что это
BIVREST генерирует уникальный тип данных: каждое действие участника получает бинарную метку «Верю / Не верю» от группы (или от ИИ-верификатора).

Это ground truth для обучения:

детекторов конгруэнтности ролевому поведению

моделей распознавания эмоций

систем поведенческой аналитики

AI-верификаторов ролевого соответствия

классификаторов убедительности

2. Формат данных
Лог одной сессии включает следующие поля (унифицировано со спецификацией):

Поле	Тип	Описание
session_id	string	Уникальный идентификатор сессии
timestamp	datetime	Дата и время хода (ISO 8601)
round	integer	Номер раунда
player_id	string	Анонимный ID игрока
avatar	string	Роль, которую играет участник
task	string	Текст задания
action_text	text	Действие игрока (вербальное или описание телесного)
votes_believe	integer	Количество голосов «Верю»
votes_dont_believe	integer	Количество голосов «Не верю»
verdict	string	Итог: «Верю» или «Не верю»
tokens_lost	integer	Количество потерянных фишек (штраф)
empty_chair	boolean	Был ли активирован «Пустой стул» (true/false)
notes	text	Дополнительные примечания (опционально)
Пример в формате CSV (шапка + данные)
csv
session_id,timestamp,round,player_id,avatar,task,action_text,votes_believe,votes_dont_believe,verdict,tokens_lost,empty_chair,notes
S001,2026-06-25T12:00:00,1,P001,Красная Шапочка,Расскажи о секретах аватара,"Она стеснительная, шепчет деревьям свои фантазии",3,1,Верю,0,false,
S001,2026-06-25T12:05:00,2,P002,Элитная проститутка,Сделай провокационное предложение,"Ты выглядишь дорого. Очень дорого",4,0,Верю,0,false,
S001,2026-06-25T12:10:00,3,P001,Красная Шапочка,Соблазни волка,"Милый волк, какой у тебя длинный... нос",1,4,Не верю,1,false,Группа сочла не соответствующим роли
S001,2026-06-25T12:20:00,4,P003,Волк,Реагируй на провокацию,"Я за себя не ручаюсь...",2,3,Не верю,2,true,Выход из роли
3. Рекомендации по сбору
Анонимизация
Заменяйте реальные имена участников на P001, P002, P003

Не сохраняйте никакой идентифицирующей информации

Полнота
Фиксируйте все ходы, включая «провальные» (саботаж, отказ, смех)

Даже если игрок выпал из роли — записывайте

Таймстемпы
Указывайте время каждого хода для анализа динамики

Контекст
Кратко описывайте невербальные действия в поле notes или в action_text в квадратных скобках

Пример:

text
[Смеётся, краснеет, отворачивается] "Ну я не знаю..."
4. Использование для обучения AI
Задача 1. Классификация конгруэнтности («Верю / Не верю»)
Обучите модель предсказывать вердикт группы по тексту action_text и avatar.

text
Вход: аватар, задание, действие
Выход: «Верю» / «Не верю»
Пример данных:

csv
text,role,task,label
"Она стеснительная, шепчет деревьям свои фантазии","Красная Шапочка","Расскажи о секретах аватара","believe"
"Милый волк, какой у тебя длинный... нос","Красная Шапочка","Соблазни волка","dont_believe"
Задача 2. Детекция саботажа и выхода из роли
Обучите модель распознавать:

Тип	Признаки
Имитация	Игрок отмазывается, не входит в роль
Уклонение	Слишком общие фразы, шутки, перевод темы
Несоответствие аватару	Действие не соответствует ожиданиям от роли
Эти случаи часто сопровождаются:

verdict = "Не верю"

tokens_lost > 0

empty_chair = true

Задача 3. Распознавание эмоций (расширенная разметка)
Для этой задачи разметьте действия по дополнительным шкалам:

Шкала	Диапазон	Описание
Стыд	0–10	Насколько действие содержит стыд/смущение
Страх	0–10	Насколько действие содержит страх/тревогу
Возбуждение	0–10	Насколько действие содержит сексуальное или эмоциональное возбуждение
Вовлечённость	0–10	Насколько игрок погружён в роль
Пример разметки:

csv
action_text,avatar,task,label,shame,fear,arousal,engagement
"Она стеснительная, шепчет деревьям свои фантазии","Красная Шапочка","Расскажи о секретах",believe,6,2,1,8
"Ты выглядишь дорого. Очень дорого","Элитная проститутка","Сделай комплимент",believe,1,1,7,9
5. Форматы для датасета
5.1. CSV (базовый)
csv
text,role,task,label,session_id,round
"Она стеснительная, шепчет деревьям свои фантазии","Красная Шапочка","Расскажи о секретах аватара","believe","S001",1
5.2. JSONL (для LLM)
jsonl
{"text": "Она стеснительная, шепчет деревьям свои фантазии", "role": "Красная Шапочка", "task": "Расскажи о секретах аватара", "label": "believe", "session_id": "S001", "round": 1}
5.3. Расширенный JSONL (с эмоциональными шкалами)
jsonl
{"text": "Она стеснительная, шепчет деревьям свои фантазии", "role": "Красная Шапочка", "task": "Расскажи о секретах аватара", "label": "believe", "shame": 6, "fear": 2, "arousal": 1, "engagement": 8}
6. Контроль качества аннотации
Рекомендации
Метод	Описание
Двойная аннотация	Два аннотатора размечают одни и те же данные
Расчёт согласованности	Каппа Коэна > 0.8 — хороший уровень
Экспертная проверка	Третий эксперт проверяет спорные случаи
Чек-лист аннотатора
Перед сохранением проверьте:

action_text содержит полный отыгрыш

avatar соответствует роли из лога

task соответствует заданию

label соответствует вердикту (believe / dont_believe)

tokens_lost — число, а не "да/нет"

empty_chair — true или false, а не "да/нет"

7. Этические требования
Информированное согласие

Данные собираются только с согласия участников

Участник может удалить свой лог в любой момент

Конфиденциальность

Данные не передаются третьим лицам без согласия

Все данные анонимизируются

Запрещённое использование

Категорически запрещено использовать данные для:

создания систем оценки «сексуальной девиантности»

дискриминационных целей

публичного раскрытия идентичности участников
