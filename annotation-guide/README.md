# Гайд по аннотации данных для AI

**Автор метода:** Олеся Лазарева

---

## 1. Что это

BIVREST генерирует уникальный тип данных: каждое действие участника получает бинарную метку «Верю» / «Не верю» от группы или ИИ.  
Это **ground truth** для обучения:

- детекторов правды / лжи
- моделей распознавания эмоций
- систем поведенческой аналитики
- AI-верификаторов ролевого соответствия

---

## 2. Формат данных

Лог одной сессии (игры) должен включать следующие поля:

- `session_id` — уникальный идентификатор сессии
- `timestamp` — дата и время хода
- `player_id` — анонимный ID игрока
- `avatar` — роль, которую играет участник
- `task` — текст задания
- `action` — действие игрока (вербальное или описание телесного)
- `votes_trust` — количество голосов «Верю»
- `votes_not_trust` — количество голосов «Не верю»
- `verdict` — итог: «Верю» / «Не верю»
- `penalty` — штраф (да / нет)
- `empty_chair` — был ли активирован пустой стул (да / нет)
- `new_script` — новый сценарий, если сформирован (текст)

### Пример в формате CSV (шапка)

```csv
session_id,timestamp,player_id,avatar,task,action,votes_trust,votes_not_trust,verdict,penalty,empty_chair,new_script
S001,2026-04-01T12:00:00,P001,Красная Шапочка,Расскажи о сексуальной жизни аватара,"Она стеснительная, шепчет деревьям свои фантазии",3,1,Верю,Нет,Нет,
S001,2026-04-01T12:05:00,P002,Элитная проститутка,Сделай сексуальный комплимент,"Ты выглядишь дорого. Очень дорого",4,0,Верю,Нет,Нет,
3. Рекомендации по сбору
Анонимизация. Заменяйте реальные имена на P001, P002.

Полнота. Фиксируйте даже «провальные» ходы (саботаж, отказ, смех).

Таймстемпы. Указывайте время каждого хода для анализа динамики.

Контекст. Кратко описывайте невербальные действия: «смеётся», «краснеет», «отворачивается».

4. Использование для обучения AI
Задача 1. Верификация «Верю / Не верю»
Обучите модель предсказывать вердикт группы по тексту action и avatar.

Вход: аватар, задание, действие
Выход: «Верю» / «Не верю»

Задача 2. Детекция саботажа
Обучите модель распознавать:

имитацию (игрок отмазывается, не входит в роль)

уклонение (слишком общие фразы, шутки)

несоответствие аватару

Задача 3. Распознавание эмоций
Разметьте действия по шкалам:

стыд (0–10)

страх (0–10)

возбуждение (0–10)

смех / веселье (0–10)

5. Этические требования
Данные собираются только с информированного согласия участников.

Участник может удалить свой лог в любой момент.

Данные не передаются третьим лицам без согласия.

Категорически запрещено использование для:

создания систем оценки «сексуальной девиантности»

дискриминационных целей

публичного раскрытия идентичности участников

6. Коммерческое использование
